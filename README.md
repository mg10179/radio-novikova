# Radio Novikova

Мобильное веб-приложение для прослушивания музыкального каталога. Хостится на GitHub Pages, MP3-файлы лежат в том же репозитории. Без бэкенда, без баз данных, без OAuth.

## Структура

```
radio-novikova/
├── index.html          ← приложение
├── albums.json         ← каталог: метаданные альбомов и треков
└── albums/
    ├── stabat-mater-rv-621/
    │   ├── cover.jpg              ← обложка (если есть)
    │   ├── 02_II__Cuius_animam_1stEdit.mp3
    │   └── ... другие треки
    └── nisi-dominus-rv-608/
        ├── cover.jpg
        └── ... треки
```

## Деплой — 5 минут

1. **Создайте GitHub-репозиторий** `radio-novikova` (публичный — Pages бесплатно работают только для публичных репозиториев на тарифе Free).

2. **Залейте файлы.** Самый простой способ — drag-and-drop:
   - На странице репозитория: **Add file → Upload files**
   - Перетащите всё содержимое архива (включая папку `albums`)
   - **Commit changes**
   
   *Альтернатива через командную строку:*
   ```bash
   cd radio-novikova
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/ВАШ_USERNAME/radio-novikova.git
   git push -u origin main
   ```

3. **Включите GitHub Pages.**
   - **Settings → Pages**
   - **Source:** `Deploy from a branch`
   - **Branch:** `main`, папка `/ (root)` → **Save**
   - Через 1–2 минуты сайт будет доступен по адресу:
     `https://ВАШ_USERNAME.github.io/radio-novikova/`

4. **Откройте на телефоне.** Можно добавить на главный экран (Safari → Поделиться → На экран «Домой»; Chrome → меню → Добавить на главный экран). Будет работать как нативное приложение.

## Добавление альбома

1. Создайте новую папку внутри `albums/`, например `albums/mass-in-b-minor/`.
2. Положите туда MP3-файлы и опционально `cover.jpg` (или `cover.png`/`cover.webp`).
3. Откройте `albums.json` в браузере на GitHub, нажмите карандаш «Edit», добавьте новую запись:
   ```json
   {
     "id": "mass-in-b-minor",
     "title": "Mass in B minor",
     "composer": "J. S. Bach",
     "artist": "Ekaterina Novikova, Symphonia Berlin Ensemble",
     "year": "2026",
     "folder": "mass-in-b-minor",
     "cover": "cover.jpg",
     "tracks": [
       { "file": "01_Kyrie.mp3",  "name": "Kyrie eleison" },
       { "file": "02_Gloria.mp3", "name": "Gloria in excelsis" }
     ]
   }
   ```
4. **Commit changes**. Сайт обновится сам через 1 минуту. В приложении нажмите кнопку ↻ — каталог перечитается.

## Удаление альбома

Уберите запись из `albums.json` (папку с файлами можно оставить или удалить — на работу сайта не влияет).

## Возможности приложения

- **Альбомы** — список всех альбомов с обложками и треклистом
- **Любимое** — ♥ рядом с треком добавляет его в Любимое. Хранится локально в браузере (не синхронизируется между устройствами).
- **Радио** — непрерывное проигрывание случайных треков из всех альбомов
- **Перемешать / Повтор** — стандартные элементы плеера
- **Управление с lockscreen** — на iOS/Android трек отображается в Control Center и на экране блокировки, кнопки play/pause/next/prev работают
- **Перемотка** — тап или драг по полосе прогресса

## Лимиты GitHub Pages

- 100 МБ на файл (с запасом для MP3 320 kbps — самый большой ваш файл ~11 МБ)
- 1 ГБ репозиторий (текущий ~84 МБ, есть место под ~10 ещё таких альбомов)
- 100 ГБ/месяц трафика (хватает на тысячи прослушиваний)

Если когда-нибудь упрётесь в эти лимиты — переезд на Cloudflare R2 или Backblaze займёт час.

## Возможные проблемы

**Сайт открывается, но «Не удалось загрузить каталог»** — проверьте, что `albums.json` действительно лежит в корне репозитория и что у него правильный синтаксис JSON. Откройте `https://ВАШ_USERNAME.github.io/radio-novikova/albums.json` напрямую в браузере — должен показаться JSON-текст.

**Треки не играют, в DevTools видна 404-ка** — путь в `albums.json` не совпадает с реальным именем файла. Имена чувствительны к регистру: `Cover.JPG` ≠ `cover.jpg`.

**Хочу свой домен** (например, `radio.example.com`) — в **Settings → Pages → Custom domain** введите домен, и пропишите CNAME на `ВАШ_USERNAME.github.io` у регистратора. Через несколько минут заработает с автоматическим HTTPS.

## Обложки

В архиве нет обложек — добавьте их сами. Положите файл `cover.jpg` (или `.png`/`.webp`) в папку альбома; рекомендуемый размер 600×600 или 800×800 пикселей. Если обложки нет — будет показана заглушка ♪ на тёмном фоне.
